# Carbon + Svelte sources — migration reference

> Canonical reference for the Argos IBM Carbon migration (**ADR 0006** — full IBM Carbon adoption as the sole UI system, IBM Plex type, zero Lunaris/custom/shadcn; reverses ADR 0001/0002). Grounded in audits of the 10 official Carbon/Svelte sources, cross-checked against the live Argos tree on `2026-05-29`.
>
> Use this to (a) cite the right upstream doc/package in the plan, (b) know exactly which package each phase depends on, and (c) avoid the verified gotchas (token-exposure trap, dispatch-vs-callback, separate icon/chart/font packages, SSR-breaks-charts).

## 2. Per-source detail

### Carbon docs site (carbondesignsystem.com)
- **Purpose:** Official Carbon documentation website. The audited page (`/contributing/documentation/`) is the *authoring* meta-guide — it defines the four standard tabs every component/pattern page uses: **Usage / Style / Code / Accessibility**. It is not itself a token reference.
- **Key value:** The four-tab structure is the lens for reading any component's contract. Real token values live on sibling pages: `/elements/color/tokens/` (color-token table) and each component's **Code** tab (carbon-components-svelte props/events) and **Accessibility** tab.
- **Argos use / phase:** Cross-cutting; primary value in **P4** when reading each component's props/events/a11y contract before swapping.
- **Svelte 5 compat:** n/a (docs).
- **Gotchas:** The audited page is meta-docs, not a token reference — pull actual `--cds-*` values from `/elements/color/tokens/` and per-component Code tabs.

### Carbon monorepo (`@carbon/styles`, `/themes`, `/colors`, `/layout`, `/type`)
- **Purpose:** IBM's design-token primitives. `@carbon/colors` = raw IDL swatches (`$blue-50`, `$gray-100`); `@carbon/themes` = the four theme token sets (white/g10/g90/g100) mapping semantic tokens → colors, emitting `--cds-*`; `@carbon/type` = IBM Plex families/weights/scale; `@carbon/layout` = spacing scale + breakpoints; `@carbon/styles` = the SCSS aggregator that compiles all of the above and emits the full `--cds-*` custom-property layer.
- **Key APIs / tokens:** SCSS theming mixin — `@use '@carbon/styles/scss/themes';` then `@use '@carbon/styles/scss/theme' with ($theme: themes.$g100)` (or a custom `$theme` map). Granular entries `scss/reset`, `scss/grid`, `scss/components/<name>`. JS exports: `white/g10/g90/g100`, `themes` map, `buttonTokens`/`tagTokens`, `formatTokenName`. Token surface = `--cds-background`, `--cds-text-primary`, `--cds-layer`, `--cds-spacing-05`, etc.
- **Argos use / phase:** Authoritative for **P2/P3** if Argos moves from precompiled CSS to compiling `@use '@carbon/styles/scss/theme'` — but note P1/P2 already shipped using the **precompiled** ccs CSS bundle, not these scopes (see §5). Adding `@carbon/styles` would let P3 drive the 13 palettes via the theme mixin (custom `$theme` maps from `@carbon/colors`) rather than hand-written overrides.
- **Svelte 5 compat:** Framework-agnostic Sass/JS — no Svelte peer at all. Only peer is `sass ^1.33.0` (optional).
- **Gotchas:** (1) These scopes are **NOT** pulled by carbon-components-svelte v0.x (it ships precompiled CSS; runtime deps are only `@ibm/telemetry-js` + `flatpickr`) — adding them is an explicit decision. (2) **Version-line mismatch:** `@carbon/styles` is on a `1.x` line while themes/colors/layout/type are `11.x` — pin each, don't assume a shared major. (3) Needs a Sass compiler. (4) `ibmtelemetry` postinstall collects metrics — opt out for the Army (`us-army-2k`) context. (5) If compiled alongside ccs's bundled CSS, keep versions aligned or `--cds-*` values diverge.

### carbon-components-svelte (`0.107.1`)
- **Purpose:** The official Svelte implementation of Carbon — a batteries-included component library (Button, DataTable, Modal, TextInput, Dropdown, ComboBox, UIShell, Tile, Toggle, Loading…) plus prebuilt theme stylesheets. Ships **both** the components and the Carbon token CSS Argos needs. Single maintainer (metonym), Apache-2.0, built-in TS types.
- **Key APIs / tokens:** Component barrel `carbon-components-svelte`. **Token CSS** (the migration crux) under `carbon-components-svelte/css/*`: `white.css`, `g10.css`, `g80.css`, `g90.css`, `g100.css`, and `all.css`. The five fixed files bake one theme; **`all.css` carries every theme + enables runtime switching via `html[theme="…"]` and is what exposes `--cds-*` for client-side theming.** Runtime `<Theme>` component + LocalStorage/SessionStorage persistence. IBM Plex `@font-face` points at the IBM CDN (`1.www.s81c.com/common/carbon/plex/…`).
- **Argos use / phase:** The library Argos already runs (27 src files import the barrel). **P1** = swap `g100.css` → `all.css` (+ `html[theme]` / mount `<Theme>`) so `--cds-*` is exposed. **P4** = swap bespoke components for ccs components; remove the bridge; remove shadcn `ui/*`. (Note: the shipped P1 used a different concrete mechanism — see §5.)
- **Svelte 5 compat:** Compatible with Svelte 3/4/**5** from one codebase. **No `peerDependencies` block** — installs cleanly against Argos's exact `svelte 5.55.7` but npm will NOT warn on mismatch; verify manually. Runs in legacy/compat mode, **not** runes: components still use `createEventDispatcher` (verified in **58** `.svelte` files — Toggle, Theme, ComboBox, ComposedModal, UIShell/HeaderSearch…).
- **Gotchas:** (1) **Dispatch vs callback props (load-bearing):** ccs emits Svelte events — consume with `on:eventname`, NOT runes callback props. Works under Svelte 5 in compat mode only. (2) Several form components historically fired a spurious initial `change`/`update` event under Svelte 5; fixed in recent `0.106.x`/`0.107.x` — the `0.107.0 → 0.107.1` bump is worth taking. (3) Icons/pictograms/charts are **separate packages**, not bundled. (4) `g100.css` alone does NOT expose cross-theme `--cds-*` — need `all.css` for dynamic theming. (5) Plex loads from the IBM CDN — needs `font-src`/`style-src` CSP allowlist for `1.www.s81c.com` (already pinned in `security-headers.ts`) or self-host. (6) ~600KB unoptimized CSS — use `optimizeImports` + `optimizeCss` (already wired). (7) `ibmtelemetry` postinstall.

### carbon-icons-svelte (`13.11.0`)
- **Purpose:** The entire Carbon SVG icon set as individual Svelte components (one `.svelte` per icon), zero-runtime-dependency, generated from `@carbon/icons`. Canonical replacement for ad-hoc icons.
- **Key APIs / tokens:** Per-icon default exports — `import Add from "carbon-icons-svelte/lib/Add.svelte"` or barrel `import { Add } from "carbon-icons-svelte"` (rewritten by `optimizeImports`). Optional props: `size` (16|20|24|32|number, default 16), `title`. `$$restProps` forwarded to `<svg>` (so `fill`, `class`, `aria-label`, `tabindex` pass through). TS type `CarbonIconProps`. **Icons carry no tokens** — color comes from CSS `fill`/`currentColor`, so they inherit the active Carbon theme automatically.
- **Argos use / phase:** **P4** — adopt as the icon standard, replacing hand-rolled icons. Set `fill` via `currentColor`/`--cds-*` so icons pick up whichever of the 13 palettes is active. Use per-icon `/lib/<Name>.svelte` paths or `optimizeImports`.
- **Svelte 5 compat:** No hard peer pin; TS support gated by major. **v13.x** uses the modern `Component` type, requires Svelte 4 or 5 — correct line for Argos. (v12.13.0 is the Svelte-3 floor.)
- **Gotchas:** (1) Separate package, not installed. (2) **No event API at all** — the dispatch-vs-callback hazard does NOT apply. (3) Barrel import pulls thousands of icons — always use `/lib/<Name>.svelte` or `optimizeImports`. (4) v13.0.0 made `CarbonIconProps` a type alias (was interface) — can't be `extends`-ed.

### carbon-pictograms-svelte (`13.17.0`)
- **Purpose:** Full Carbon **pictogram** set (large illustrative line-art, distinct from UI icons) as zero-dependency Svelte components. For empty states, onboarding, section headers.
- **Key APIs / tokens:** `import Airplane from "carbon-pictograms-svelte/lib/Airplane.svelte"`. Only prop is `title?`; `$$restProps` spread onto `<svg>` (fill/class/width/height). Default fill `currentColor`, default size 64×64. **No `size` prop** — control via width/height. TS type `CarbonPictogramProps`. Token-transparent (inherits `color`).
- **Argos use / phase:** **P4, optional/low-priority** — only if Argos has illustrative empty/onboarding surfaces. Token-transparent, so it doesn't block P1/P2.
- **Svelte 5 compat:** v13.x ready (Svelte 4/5, modern `Component` type). No events, no slots — nothing to migrate.
- **Gotchas:** (1) Separate package, not installed. (2) Pictograms ≠ icons (different package). (3) Barrel import is huge — use `/lib/Name.svelte` or `optimizeImports`. (4) Fast-moving minor bumps tracking `@carbon/pictograms` upstream. (5) Type alias since 13.0.0.

### @carbon/charts-svelte (`next` = `1.27.11`)
- **Purpose:** 26 chart types for Svelte 5.20+, a thin wrapper over the vanilla/D3 `@carbon/charts` core. Renders against Carbon tokens, ships prebuilt CSS, auto-matches themes/categorical palettes. The canonical home for Argos's deferred `--signal-*`/`--chart-*`/`--feature-*` domain colors.
- **Key APIs / tokens:** Exports the 26 chart components (`BarChartSimple`, `LineChart`, `DonutChart`…) + enums/types re-exported from `@carbon/charts`. **Theming via the `options` prop:** `options.theme` (`white|g10|g90|g100`), `options.color.scale`/`options.color.pairing` map data groups onto Carbon categorical tokens (or override `--cds-charts-*` CSS props). **Required CSS:** `@carbon/charts-svelte/styles.css`. Type props via `ComponentProps<typeof BarChartSimple>`.
- **Argos use / phase:** **P4** (deferred out of P2 into P4). Add `@carbon/charts-svelte@next` + locked `@carbon/charts` core; import `styles.css`; feed `data` + `options` whose `theme` matches P1/P2 and whose `options.color.scale` maps signal/feature groups onto the **P3** palette. **Cannot land before P3** (it consumes the palette work).
- **Svelte 5 compat:** **Svelte-5 native** — the `next`/1.27.11 line declares `peerDependencies { svelte: ^5.20.2 }` (Svelte 5 only; Argos's 5.55.7 satisfies). **Fully callback-props** — NO `createEventDispatcher`. Lifecycle via callback props `{onload}` `{onupdate}` `{ondestroy}`; chart instance via `bind:chart` (`$state`), low-level events via `chart.services.events.addEventListener(...)` in `onMount`. No migration debt.
- **Gotchas:** (1) Separate package + locked core (pulls its own D3); keep `charts-svelte` and `@carbon/charts` on the **same exact** version. (2) **Tag trap:** plain `npm i @carbon/charts-svelte` gets `1.22.18` (broad-peer latest), NOT the Svelte-5-tuned `1.27.11` — README mandates `@next`. (3) **SSR WILL BREAK** — `@carbon/charts` needs `window`. Argos is SvelteKit/SSR, so mandatory: dynamic-import inside `onMount`, OR `vite.config` `ssr: { noExternal: [...] }`. (4) Harmless d3 circular-dependency build warnings. (5) `ibmtelemetry` postinstall — opt out for air-gapped/Army deploy. (6) `styles.css` import required or charts render unstyled. (7) Two-tier events: high-level = callback props, granular hover/click = imperative `chart.services.events` after mount.

### carbon-preprocess-svelte (`0.11.30`)
- **Purpose:** Zero-dependency build tooling for Carbon. `optimizeImports` (Svelte preprocessor) rewrites barrel imports to source-file paths; `optimizeCss` (Vite plugin, build-only) strips unused `bx--` selectors and unused Plex `@font-face` rules. README example: 606 kB → 53 kB (−91%).
- **Key APIs / exports (corrected — task brief was wrong):** Exactly **three** exports in 0.11.30 — `optimizeImports` (→ `svelte.config` `preprocess[]`), `optimizeCss` (→ vite `plugins[]`, `apply:"build"`), and `OptimizeCssPlugin` (Webpack-class equivalent). **There are NO `Theme`, `collect`, or `elements` exports** — those were legacy 0.1.x preprocessors, long removed. `optimizeCss` options: `{ silent?, preserveAllIBMFonts? }` (default false — keeps only Plex Sans 300/400/600 + Mono 400). **Emits no tokens/themes/fonts itself.**
- **Argos use / phase:** Already wired — `svelte.config.js:10` (`preprocess: [vitePreprocess(), optimizeImports()]`) and `vite.config.ts:49` (`optimizeCss()`). Cross-cutting; most directly supports **P4** (every component/icon import benefits) and the CSS-size side of **P1/P5**. No role in P2/P3.
- **Svelte 5 compat:** Build-time AST/CSS transform — dispatch-vs-callback is irrelevant. Dev-pins `svelte ^5.55.7` (exact match to Argos) + ships a `webpack@svelte-5` example.
- **Gotchas:** (1) **Brief's `Theme`/`collect`/`elements` exports do not exist** — don't plan to import them. (2) `optimizeImports` is a preprocessor slot, `optimizeCss` is a Vite-plugin slot — don't mix. (3) `optimizeCss` runs **only on production build** — no dev pruning. (4) Preprocess order: `vitePreprocess` (TS) must run **before** `optimizeImports`. (5) **`preserveAllIBMFonts` defaults false** — drops most Plex faces; if **P5** needs other weights/families (Plex Serif, Sans 700) set `preserveAllIBMFonts:true` or lose them in the build. (6) `prebundleSvelteLibraries` now defaults true, so `optimizeImports` dev-speed gain is marginal; the real win is `optimizeCss` in production.

### Svelte docs / Svelte repo (`svelte` `5.55.10`)
- **Purpose:** The compiler + runtime under SvelteKit. Ships **no design tokens** — but is the substrate every migration edit lands on. Docs are the authority on runes, snippets/render tags, the new event model, and legacy-to-5 interop.
- **Key APIs:** Runes (`$state`, `$derived`/`.by`, `$props`, `$effect`, `$bindable`, `$inspect`). `{#snippet}` + `{@render}` replace `<slot>`; children via `children` prop. Events: `onclick={fn}` attribute replaces `on:click`; **`createEventDispatcher` is deprecated** → callback props (`let { onfoo } = $props()`). Module API `mount`/`hydrate`/`unmount`; legacy shims under `svelte/legacy`.
- **Argos use / phase:** Cross-cutting authoring authority; matters **most in P4** when wiring ccs components — consume ccs's `on:eventname` events under Svelte 5 compat, author NEW Argos code with callback props + `onclick`. Pair with the `tessl__svelte-best-practices` skill on every component edit.
- **Svelte 5 compat:** This IS Svelte. Argos on `5.55.7` (pinned exact, no caret), 3 patches behind latest `5.55.10` — same minor, safe bump; do deliberately before P4.
- **Gotchas:** (1) Ships no `--cds-*`/Plex — don't conflate. (2) `createEventDispatcher` deprecated but functional in 5.55.x — don't add new ones; **the interop trap is mixing ccs's dispatch events with runes callback-prop expectations**. (3) Duplicate `on:click` now illegal — combine into one `onclick`; when spreading props, local handlers must come AFTER the spread. (4) `bind:` not free in runes — child prop must be `$bindable()`; bites when two-way-binding Carbon form inputs. (5) Slots → snippets/`{@render}`; root layout wrapping `<Theme>` must use `{@render children()}`. (6) npm `next` dist-tag (`5.0.0-next.272`) is a stale prerelease — ignore.

### SvelteKit (`@sveltejs/kit` `2.61.1`)
- **Purpose:** The application framework hosting Argos's v1 UI. Owns routing, `+layout`/`+page`/`+server`/`hooks` conventions, SSR/hydration/prerender, the Vite plugin, and adapters. For the migration it's the host: where global CSS, the `<Theme>` provider, IBM Plex, and SSR theming mount.
- **Key surfaces (conventions, not runtime exports):** (1) **root `src/routes/+layout.svelte`** — single mount for Carbon global CSS + `<Theme>` wrapping `{@render children()}`. (2) `src/app.html` — Plex `<link>`/preload (P5). (3) `src/app.css` / global stylesheet — the token bridge maps Lunaris → `--cds-*` (P2/P3). (4) `@sveltejs/kit/vite` `sveltekit()` plugin beside `optimizeImports`/`optimizeCss`. (5) **`src/hooks.server.ts` `transformPageChunk`** — the supported way to stamp `data-theme`/class onto server-rendered HTML for **flash-free SSR theming** (the key P3 lever).
- **Argos use / phase:** Threads through every phase via convention files. **P1** = the one-line CSS import in `+layout.svelte`. **P2/P3** = the global bridge stylesheet + `<Theme>` + `transformPageChunk` theme cookie. **P4** = route-level code-splitting governs Carbon chunking — keep imports granular. **P5** = Plex into `app.html`.
- **Svelte 5 compat:** Fully compatible. Peers `svelte ^4.0.0 || ^5.0.0-next.0`; Argos's vite 7.3.3 / vite-plugin-svelte 6.0.0 / svelte 5.55.7 all satisfy. Installed `^2.60.1`, latest `2.61.1` (in range).
- **Gotchas:** (1) Adapters + the Vite plugin are **separate packages**. (2) SvelteKit carries **no Carbon tokens** — only mount points. (3) **Theme/SSR flash:** client-only theme in `+layout` flashes wrong theme on first paint — fix via `hooks.server.ts` `transformPageChunk` (the single most important SvelteKit gotcha for P3). (4) `createEventDispatcher` is a Svelte, not SvelteKit, concern — no dispatch API here. (5) Root layout must use `{@render children()}` under Svelte 5. (6) 2.61.0 removed `.run()` on remote queries (irrelevant to UI, but note when bumping the lock past 2.60.1).

## 5. Status reflection — current progress

**P1 (expose tokens) — DONE (PR #281).** Verified live via chrome-devtools / Sentry / otel. The migration-context goal was to make `--cds-*` resolve app-wide. The shipped mechanism, confirmed in the live tree, is the **root import `carbon-components-svelte/css/g100.css` plus `theme="g100"` on `<html>`** — after which **457 `--cds-*` custom properties resolve**.
